# Corpus — sources the AI chat widget is grounded on

This folder holds the **only** material the council-watch chat widget can cite. The widget calls OpenAI's Responses API with the `file_search` tool against a vector store built from these files. If a fact isn't in this folder, the AI is instructed to refuse rather than guess.

## Layout

- `index.json` — the manifest. Every entry has an `id`, a `type` (`local` or `url`), a human `title`, and a `category`. URL entries carry a `format` hint (`html`, `pdf`, `skip`).
- `<id>.md` — cached, human-readable extract of a URL entry (HTML format). Each file starts with YAML frontmatter recording `source_url`, `fetched_at`, `title`, and `category`. **Committed to git** so corpus changes are reviewable in PRs.
- `<id>.pdf` — raw download of a PDF source. Uploaded directly to OpenAI; `file_search` parses PDFs natively.
- `file-map.json` — generated by the indexer. Maps the OpenAI `file_id` returned at upload back to the corpus `id` + title + canonical URL. The Worker reads this at runtime to turn `file_citation` annotations into clickable source links in the chat panel.

Local-file entries (HTML pages elsewhere in the repo, `PRIMARY-SOURCES.md`) are read in place — the indexer doesn't copy them here. HTML pages have their `<header>` and `<footer>` stripped before upload to reduce template noise in retrieval.

## Rebuilding

```bash
# One-time, before first run:
wrangler secret put OPENAI_API_KEY
# (Then for local indexing only, also export to your shell:)
export OPENAI_API_KEY=...

# Each time a source changes (URL refreshed, new entry added, local page edited):
npm run build-corpus
```

The indexer is idempotent: it computes a content hash per source, only re-uploads changed files, and removes vector-store files that no longer appear in `index.json`. It also writes the up-to-date `file-map.json`.

After the first run, the script will print the OpenAI vector store ID. Paste that into `wrangler.jsonc` `vars.VECTOR_STORE_ID`.

## Editorial discipline

Only material we can verify against a primary source belongs in this corpus. See `../learnings.md` and `../CLAUDE.md` for the longer story on why. AI summaries, preview articles, and second-hand reporting are flagged with `category: "press"` so the AI knows to attribute carefully.
